测试方案总览

核心思路：程序在 FPGA 上运行特定指令访问模式，I-Cache 的命中/未命中计数器通过两种方式输出：

方法所需设备显示内容精度
A: 数码管 + LED仅需板子命中/未命中次数 (32位十六进制)精确计数
B: UART 串口板子 + USB 线 + PC命中/未命中 delta + 命中率精确 + 自动计算

两种方法可以同时使用，互不冲突。

观察原理

开关设置数码管显示说明
SW[15:14] = 00PC 值原始调试模式
SW[15:14] = 01hit_countI-Cache 命中次数 (从复位开始累计)
SW[15:14] = 10miss_countI-Cache 未命中次数
SW[15:14] = 11hit + miss总访问次数

LED[15:0] 显示命中率 = hit_delta × 1000 ÷ (hit_delta + miss_delta)，范围 0-1000，对应 0.0%-100.0%。

# 预期资源使用：

步骤 3: 烧录到板子

5

连接板子

  1. 用 USB 线连接 Nexys4 DDR 的 PROG/UART 端口（J6）到电脑
  2. 打开板子电源开关（POWER 开关拨到 ON）
  3. 确认板子上的 DONE LED 亮起（表示 FPGA 配置完成，如果之前烧过）
  4. FPGA MODE 跳线设为 JTAG（M[2:0] = 000）
6

烧录 Bitstream

  1. 打开 Vivado Hardware Manager
  2. Open Target → Auto Connect（应看到 xc7a100t）
  3. 右键器件 → Program Device → 选择生成的 .bit 文件
  4. 点击 Program，等待烧录完成

烧录后，FPGA 立即开始运行测试程序。程序执行完成后会进入死循环，LED 和数码管显示最终结果。

观察方法 A: LED 数码管

读取命中率 (LED)

程序运行结束后，LED[15:0] 显示命中率：

LED 显示值 (十六进制)十进制命中率
0x3E81000100.0%
0x3E099299.2%
0x3D097697.6%
0x35084884.8%
0x00000.0%

换算公式：命中率 = LED值 ÷ 10 (百分比)

读取精确命中/未命中次数 (数码管)

通过拨码开关 SW[15:14] 切换数码管显示内容：

看命中次数

  1. 将 SW[15] 拨到下方 (0)，SW[14] 拨到上方 (1)
  2. 数码管显示 8 位十六进制数 = hit_count
  3. 例如显示 00003E80 = 16000 次命中

看未命中次数

  1. 将 SW[15] 拨到上方 (1)，SW[14] 拨到下方 (0)
  2. 数码管显示 8 位十六进制数 = miss_count
  3. 例如显示 00000050 = 80 次未命中

看总访问次数

  1. 将 SW[15] 和 SW[14] 都拨到上方 (1)
  2. 数码管显示 hit_count + miss_count

手动计算命中率：hit_count ÷ (hit_count + miss_count) × 100%

测试程序说明

1. 紧凑循环 (test_loop.hex)

访问模式：4 条 NOP 指令的循环体，执行 10000 次

地址范围：仅约 20 字节的循环体，完全在 8 路 cache 的同一组中

预期结果：第一次迭代可能 miss（冷启动），之后全部 hit。命中率 ≈ 99.9%+

测试意义：验证 PMRU 的 temporal locality 保护能力

2. 顺序执行 (test_sequential.hex)

访问模式：512 条连续 NOP 指令，PC 线性递增

地址范围：2048 字节，与 cache 容量相同

预期结果：流式预取提前加载指令，流式旁路在 16 次顺序访问后跳过 cache。命中率取决于预取命中率

测试意义：验证 16 入口流式预取 + 流式旁路的效果

3. 分支密集 (test_branchy.hex)

访问模式：交替 BEQ 跳转到相隔 64 字节的地址，执行 100 次

地址范围：跨度约 400 字节，涉及多个 cache 组

预期结果：分支目标首次 miss，后续可能 hit（如果在 cache 中）。命中率取决于分支目标分布

测试意义：验证 PMRU 对非顺序访问的处理能力

3. 混合模式 (test_mixed.hex)

访问模式：200 条顺序 + 500 次紧凑循环 + 50 次分支

地址范围：约 1000 字节，部分在 cache 中

预期结果：综合三种模式的特征，命中率应该较高

测试意义：模拟真实程序的混合访问模式

常见问题

Q: 数码管不亮 / 全灭

A: 检查 SW[15:14] 是否设置正确。设置为 00 可看 PC 值。如果 PC 值也不显示，检查 FPGA 是否烧录成功。

Q: LED 全灭 / 全亮

A: 程序可能还在运行中（未到报告代码），或者程序已跑飞。按 RESET 按钮重新运行。如果 LED 全亮 (0xFFFF)，说明除零错误，可能计数器都为 0。

Q: UART 接收不到数据

A: 依次检查：

  1. USB 线是否连接到 PROG/UART 端口 (J6)，不是 JTAG-only 端口
  2. 设备管理器中是否出现 COM 口（可能需要安装 FTDI 驱动）
  3. 波特率是否为 115200
  4. 程序是否执行到了 UART 发送代码（LED 应显示非零值）
  5. 按 RESET 重新运行程序

Q: 综合报错找不到 firmware.hex

A: 确保 firmware.hex 已复制到 synth/vivado/ 目录。run_direct.tcl 会自动将其复制到 rtl/core/。

copy "board\test_loop.hex" "synth\vivado\firmware.hex"

Q: 综合后 BRAM 使用为 0

A: 这是正常的——如果 firmware.hex 为空或不存在，Vivado 会优化掉存储器。确保 hex 文件非空且格式正确（每行一个 8 位十六进制数）。

Q: 想换一个测试程序怎么办

A: 不需要重新修改 RTL，只需要：

# 1. 复制新的 hex 文件
copy "board\test_sequential.hex" "synth\vivado\firmware.hex"

# 2. 在 Vivado 中重新综合 + 烧录
# (Run Synthesis → Run Implementation → Generate Bitstream → Program)
