# 测试2：CPU 自增计数器（Counter）

## 验证目的

验证 CPU 的顺序执行能力。程序不断对一个寄存器递增，并通过 GPIO 输出到 LED，用肉眼观察 LED 按二进制递增。相比测试1的简单闪烁，此测试验证了 ALU 加法运算、寄存器写回、以及持续的状态变化。

## 固件文件

`firmware_counter.hex`（10 条指令）

## 伪代码

```
GPIO_BASE = 0x10010000        # lui x5, 0x10010
counter = 0                   # addi x6, x0, 0

loop:
    GPIO[0] = counter          # 输出当前计数值到 LED
    delay(262144)              # ~31ms 延迟，约 16Hz
    counter = counter + 1      # 计数器递增
    goto loop                  # jal
```

## 汇编指令详解

| 地址 | 指令 | 说明 |
|------|------|------|
| 0x00 | `lui x5, 0x10010` | x5 = 0x10010000 (GPIO 基地址) |
| 0x04 | `addi x6, x0, 0` | x6 = 0 (计数器初始化) |
| 0x08 | `sw x6, 0(x5)` | GPIO = x6 (输出当前计数值) |
| 0x0C | `lui x14, 0x40` | 延迟计数器 = 262144 |
| 0x10 | `addi x14, x14, -1` | 延迟循环递减 |
| 0x14 | `bnez x14, -4` | 循环直到 x14=0 |
| 0x18 | `nop` | 延迟槽（被冲刷） |
| 0x1C | `addi x6, x6, 1` | 计数器 +1 |
| 0x20 | `j 0x08` | 跳回输出循环 |
| 0x24 | `nop` | 延迟槽（被冲刷） |

## 关键点

- **延迟时间**：262144 × 3 周期 × 40ns ≈ 31ms，更新频率约 32Hz，肉眼可见递增
- **ALU 运算**：每次循环执行 `addi x6, x6, 1`，验证 ALU ADD 功能
- **寄存器写回**：x6 递增后写回寄存器堆，验证 WB 阶段
- **二进制显示**：LED 直接显示 x6 的二进制值，0→1→2→3→...→255→0→...
- **溢出行为**：x6 到 0xFFFF 后回绕到 0x0000，LED 全灭后重新开始

## 预期结果

| 观察项 | 预期 |
|--------|------|
| LED[15:0] | 按二进制递增：0→1→2→3→...→15→...→255→0→1... |
| 递增速度 | 约 32 次/秒（每次 31ms） |
| 数码管 | 显示 0x0C / 0x10 / 0x14 循环 |

## 通过标准

- LED 明显按二进制递增模式变化
- 能观察到最低位 LED 快速闪烁，高位 LED 变化较慢
- 16 个 LED 依次全部点亮后回绕重置

## 失败排查

| 现象 | 可能原因 |
|------|---------|
| LED 不变 | ALU ADD 故障、寄存器写回失败 |
| LED 随机变化 | 取指错误、I-Cache 问题 |
| LED 递增太快/太慢 | 延迟循环计数错误 |

## 实际测试结果

在硬件上下载固件后，观察到 LED 按二进制递增模式变化：0→1→2→3→...→255→0 循环。最低位 LED 快速闪烁，高位 LED 变化较慢，递增频率约 32Hz，肉眼清晰可见。计数到 0xFFFF 后正确回绕到 0x0000 重新开始。

| 观察项 | 实际结果 |
|--------|----------|
| LED[15:0] | 按二进制递增 0→1→2→...→255→0 循环 |
| 递增速度 | 约 32Hz（每次约 31ms） |
| 回绕行为 | 0xFFFF 后正确回绕到 0x0000 |

**测试结果：PASS**
