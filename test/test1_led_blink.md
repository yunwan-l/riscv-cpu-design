# 测试1：LED 闪烁（LED Blink）

## 验证目的

验证 CPU 最基本的取指-译码-执行-访存-写回流水线能否正常工作。这是最基础的"点亮"测试，确认整个 SoC（时钟分频、CPU、GPIO 外设、LED 引脚连接）连通无误。

## 固件文件

`firmware_blink.hex`（14 条指令）

## 伪代码

```
GPIO_BASE = 0x10010000        # lui x5, 0x10010

loop:
    GPIO[0] = 0xFFFF           # 全部 LED 点亮 (sw)
    delay(4096)                # 短延迟循环
    GPIO[0] = 0x0000           # 全部 LED 熄灭 (sw)
    delay(4096)                # 短延迟循环
    goto loop                  # jal
```

## 汇编指令详解

| 地址 | 指令 | 说明 |
|------|------|------|
| 0x00 | `lui x5, 0x10010` | x5 = 0x10010000 (GPIO 基地址) |
| 0x04 | `addi x1, x0, -1` | x1 = 0xFFFF (全 1) |
| 0x08 | `addi x3, x0, 0` | x3 = 0x0000 (全 0) |
| 0x0C | `sw x1, 0(x5)` | GPIO = 0xFFFF → 16 个 LED 全亮 |
| 0x10 | `lui x10, 0x1` | 延迟计数器 = 4096 |
| 0x14 | `addi x10, x10, -1` | 延迟循环递减 |
| 0x18 | `bnez x10, -4` | 循环直到 x10=0 |
| 0x1C | `nop` | 延迟槽（被冲刷） |
| 0x20 | `sw x3, 0(x5)` | GPIO = 0x0000 → 16 个 LED 全灭 |
| 0x24 | `lui x10, 0x1` | 延迟计数器 = 4096 |
| 0x28 | `addi x10, x10, -1` | 延迟循环递减 |
| 0x2C | `bnez x10, -4` | 循环直到 x10=0 |
| 0x30 | `nop` | 延迟槽（被冲刷） |
| 0x34 | `j 0x04` | 跳回循环开始 |

## 关键点

- **延迟时间**：4096 × 3 周期 × 40ns = 0.49ms，闪烁频率约 1kHz，肉眼看到的是常亮（视觉暂留）
- **实际观察**：16 个 LED 全部同时亮灭，但由于频率太高，看起来是常亮或微闪
- **GPIO 地址**：0x10010000，SoC 地址译码 `dbus_addr[31:16] == 16'h1001` 命中 GPIO
- **时钟分频**：100MHz → 25MHz（4 分频），每周期 40ns
- **I-Cache**：循环体 4 条指令（0x10~0x1C），全部命中缓存

## 预期结果

| 观察项 | 预期 |
|--------|------|
| LED[15:0] | 16 个 LED 全部亮（因闪烁频率 ~1kHz 高于视觉暂留阈值） |
| 数码管 | 显示延迟循环 PC 值（0x14 / 0x18 / 0x1C 交替） |
| 串口 | 无输出 |

## 通过标准

- 16 个 LED 全部亮起（至少看起来是常亮）
- 系统没有死机，数码管有稳定的数字显示

## 失败排查

| 现象 | 可能原因 |
|------|---------|
| LED 全灭 | 固件未加载、GPIO 地址错误、比特流未烧录 |
| 数码管全灭 | 时钟分频问题、复位未释放 |
| 只有个别 LED 亮 | GPIO wdata 宽度不对、XDC 约束错误 |

## 实际测试结果

在硬件上下载固件后，观察到 16 个 LED 全部点亮。由于闪烁频率约 1kHz，远高于人眼视觉暂留阈值，LED 呈现为常亮状态，符合预期。系统运行稳定，数码管有正常显示。

| 观察项 | 实际结果 |
|--------|----------|
| LED[15:0] | 16 个 LED 全部亮（高频闪烁表现为常亮） |
| 闪烁频率 | ~1kHz（视觉暂留，肉眼观察为常亮） |
| 系统状态 | 运行正常，无死机 |

**测试结果：PASS**
