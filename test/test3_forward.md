# 测试3：流水线前递（Forwarding）

## 验证目的

验证 CPU 流水线的**数据前递（Forwarding）机制**。12 条连续的数据依赖指令，每条都使用前一条指令的结果，如果前递逻辑（EX/MEM 前递、MEM/WB 前递）有 Bug，最终结果就会错误。

## 固件文件

`firmware_forward.hex`（25 条指令，已修复）

## 伪代码

```
GPIO_BASE = 0x10010000        # lui x5, 0x10010

main_loop:
    x1  = 1                   # addi x1, x0, 1
    x2  = x1 + 1              # addi x2, x1, 1    ← 前递 x1 (EX/MEM)
    x3  = x2 << 1             # slli x3, x2, 1    ← 前递 x2 (EX/MEM)
    x4  = x3 << 1             # slli x4, x3, 1    ← 前递 x3
    x6  = x4 << 1             # slli x6, x4, 1    ← 前递 x4 (跳过 x5)
    x7  = x6 << 1             # slli x7, x6, 1    ← 前递 x6
    x8  = x7 << 1             # slli x8, x7, 1
    x9  = x8 << 1             # slli x9, x8, 1
    x10 = x9 << 1             # slli x10, x9, 1
    x11 = x10 << 1            # slli x11, x10, 1
    x12 = x11 << 1            # slli x12, x11, 1
    x13 = x12 << 1            # slli x13, x12, 1  → x13 = 2048 = 0x0800

    GPIO[0] = x13             # LED 11 亮
    delay(262144)             # ~31ms

    GPIO[0] = 0               # LED 灭
    delay(262144)             # ~31ms

    goto main_loop            # jal 0x04
```

## 汇编指令详解

| 地址 | 指令 | 前递链 | 期望值 |
|------|------|--------|--------|
| 0x00 | `lui x5, 0x10010` | - | x5=0x10010000 |
| 0x04 | `addi x1, x0, 1` | 无 | x1=1 |
| 0x08 | `addi x2, x1, 1` | EX/MEM 前递 x1 | x2=2 |
| 0x0C | `slli x3, x2, 1` | EX/MEM 前递 x2 | x3=4 |
| 0x10 | `slli x4, x3, 1` | EX/MEM 前递 x3 | x4=8 |
| 0x14 | `slli x6, x4, 1` | EX/MEM 前递 x4 | x6=16 |
| 0x18 | `slli x7, x6, 1` | EX/MEM 前递 x6 | x7=32 |
| 0x1C | `slli x8, x7, 1` | EX/MEM 前递 x7 | x8=64 |
| 0x20 | `slli x9, x8, 1` | EX/MEM 前递 x8 | x9=128 |
| 0x24 | `slli x10, x9, 1` | EX/MEM 前递 x9 | x10=256 |
| 0x28 | `slli x11, x10, 1` | EX/MEM 前递 x10 | x11=512 |
| 0x2C | `slli x12, x11, 1` | EX/MEM 前递 x11 | x12=1024 |
| 0x30 | `slli x13, x12, 1` | EX/MEM 前递 x12 | x13=2048=0x0800 |
| 0x34 | `sw x13, 0(x5)` | - | GPIO=0x0800 |
| 0x38~0x44 | 延迟循环 | - | ~31ms |
| 0x48 | `sw x0, 0(x5)` | - | GPIO=0 |
| 0x4C~0x58 | 延迟循环 | - | ~31ms |
| 0x5C | `j 0x04` | - | 跳回 main_loop |

## 关键点

- **EX/MEM 前递**：每条指令的源操作数来自上一条指令的 ALU 结果，尚未写回寄存器堆，必须通过 EX/MEM 前递路径获取
- **前递优先级**：`rvp_forward_unit.sv` 中 EX/MEM 优先于 MEM/WB，确保取到最新的值
- **跳过 x5**：前递链从 x4 跳到 x6（跳过 x5），因为 x5 保存 GPIO 基地址不能被覆盖，同时验证前递不会错误匹配
- **最终结果**：x13 = 2^11 = 2048 = 0x0800 → 仅 LED 11 亮
- **延迟时序**：262144 × 3 周期 × 40ns ≈ 31ms，闪烁频率约 16Hz

## 前递链数据流

```
x1=1 → x2=2 → x3=4 → x4=8 → x6=16 → x7=32 → x8=64 → x9=128 → x10=256 → x11=512 → x12=1024 → x13=2048
 2^0    2^1    2^2    2^3    2^4     2^5    2^6    2^7     2^8     2^9     2^10     2^11
```

每一步都是前一步左移 1 位，任何一个前递环节出错都会导致后续所有值错误。

## 预期结果

| 观察项 | 预期 |
|--------|------|
| LED 11 | 以约 16Hz 闪烁（亮 31ms + 灭 31ms） |
| 其他 LED | 始终熄灭 |
| 数码管（LED 亮时） | 0x3C / 0x40 / 0x44 快速交替 |
| 数码管（LED 灭时） | 0x50 / 0x54 / 0x58 快速交替 |
| 串口 | 无输出 |

## 通过标准

- **只有 LED 11 闪烁**，其他 LED 不亮
- 闪烁频率肉眼可见（约 16Hz）
- 如果前递有 Bug，x13 不等于 0x0800，LED 11 不会亮，可能其他 LED 亮或全灭

## 失败排查

| 现象 | 可能原因 |
|------|---------|
| LED 4 亮（0x10） | 前递链在某一环拿到 0 而非正确值 |
| 全灭 | 前递完全失效，x13=0 |
| 多个 LED 亮 | 前递拿到的值部分正确部分错误 |
| 全亮 | x13=0xFFFF，前递逻辑严重错误 |

## 实际测试结果

在硬件上下载固件后，观察到仅 LED 11 以约 16Hz 频率闪烁，其余 LED 始终熄灭。这与 x13 = 0x0800 = 2^11 的预期结果完全一致，证明从 x1 到 x13 共 11 级 EX/MEM 前递链路全部正确工作，数据依赖指令的流水线前递机制无误。

| 观察项 | 实际结果 |
|--------|----------|
| LED 11 | 以约 16Hz 闪烁（亮灭交替） |
| 其他 LED | 始终熄灭 |
| 前递链验证 | x1=1 → x13=2048=0x0800，11 级前递全部正确 |

**测试结果：PASS**
