## 项目概述

这是一个基于 **Nexys4 DDR (Artix-7 XC7A100T)** 的 **RISC-V RV32I** 五级流水线处理器 SoC 项目，使用 **SystemVerilog** 编写，哈佛结构，支持可选 I-Cache / D-Cache。

---

### 一、全局包与配置

#### 1. `rvp_pkg.sv` — 全局包

**实现方式：** 用 SystemVerilog `package` 集中定义所有共享类型：

- **ALU 操作码枚举 `alu_op_e`**：5 位编码，覆盖 16 种操作（10 种算术/逻辑 + 6 种分支比较）。分支和 SLT 共用同一套比较器节省硬件。
- **立即数类型枚举 `imm_type_e`**：区分 I/S/B/U/J 五种 RISC-V 立即数格式。
- **写回来源枚举 `wb_sel_e`**：ALU/MEM/PC4/IMM 四种来源供写回级选择。
- **控制信号结构体 `ctrl_t`**：把所有译码信号（寄存器地址、ALU 控制、访存控制、PC 控制、非法指令标志）打包为一个 struct，通过流水线寄存器逐级传递。
- 提供 `ctrl_zero()` 函数返回全零且 `illegal=1` 的默认控制信号。

#### 2. `rvp_config.svh` — 编译期配置

**实现方式：** 全部用 `` `ifndef `` 宏守卫+`` `define `` 形式实现，各参数可在命令行用 `+define+` 覆盖：
- ISA 配置：RV32I/E、M 扩展、C 扩展开关
- 流水线配置：5 级深度、写回级开关、专用分支 ALU 开关
- 冒险处理：前递开关（`RVP_FORWARDING`），分支预测级别（0-3）
- Cache 配置：I/D-Cache 使能、容量、路数、替换策略（0-5 共 6 种）
- 存储器配置：指令/数据存储器容量（各 32KB）
- 外设配置：UART/GPIO 使能、波特率、宽度

#### 3. `rvp_cache_pkg.sv` — Cache 参数包

**实现方式：** 用 `package` + 编译期 `$clog2` 计算所有结构参数：
- 从容量/路数/行大小自动推导 index 位宽、tag 位宽
- 定义替换策略枚举 `replacement_policy_e`：RR/LRU/PLRU/FIFO/RANDOM/SRRIP/DRRIP 共 7 种
- 定义 Cache 状态机枚举 `cache_state_e`：IDLE→LOOKUP→MISS→REFILL→WB→FLUSHING→ERROR
- 定义命中率统计结构体 `cache_stats_t`（总访问、命中、缺失、驱逐）
- I-Cache tag 只含 `valid`，D-Cache tag 额外含 `dirty` 位

### 二、CPU 核心 — 五级流水线

#### 4. `rvp_core.sv` — 核心顶层

**实现方式：** 纯结构化的模块连线层，实例化所有子模块并连接流水线级间信号：
- 参数化配置（ISA/流水线/Cache/前递），通过 `#()` 传递
- 定义 IF→ID→EX→MEM→WB 五级之间的所有数据信号（指令、PC、控制信号、操作数、ALU 结果等）
- 条件编译 `RVP_FORWARDING` 宏，控制是否实例化前递单元
- stall/flush 信号由 hazard_unit 产生，分发到各级
- 对外暴露标准总线接口（指令总线 + 数据总线）、中断接口、调试接口

#### 5. `rvp_if_stage.sv` — IF 取指级

**实现方式：**
- **PC 寄存器**：`always_ff` 时序逻辑，`boot_addr_i` 复位值
- **PC 下一值选择**：`always_comb` + `unique case`，根据 `pc_sel_i`（2 位编码）选择 boot/PC+4/分支目标/异常向量
- **PC+4 计算**：纯组合逻辑 `pc_q + 32'd4`
- **取指请求**：`instr_req_o = ~stall_i & ~flush_i`
- **IF/ID 流水线寄存器**：`always_ff` 时序逻辑，flush 时写入 NOP (`0x00000013`)，stall 时保持
- 直接取指模式下，指令数据来自总线 `instr_rdata_i`

#### 6. `rvp_id_stage.sv` — ID 译码级

**实现方式：** 模块间连线 + 子模块实例化：
- **IF/ID 流水线寄存器**：锁存指令、PC、valid
- 实例化 **`rvp_decoder`**：指令 → 控制信号结构体
- 实例化 **`rvp_imm_generator`**：根据 `imm_type` 拼出 32 位立即数
- 实例化 **`rvp_register_file`**：异步读 `rs1/rs2`，WB 级写回
- **前递选择**：条件编译，用 `unique case` 在 RF 原始值和前递数据间选择
- 输出特殊指令标志：`illegal_insn/ecall/ebreak/mret/wfi`

#### 7. `rvp_ex_stage.sv` — EX 执行级

**实现方式：**
- **ID/EX 流水线寄存器**：锁存控制信号、操作数、立即数、PC；flush 时清零 `reg_write/mem_read/mem_write`
- **操作数选择**：`alu_src_a` 决定 A=rs1 或 PC，`alu_src_b` 决定 B=rs2 或 imm
- 实例化 **`rvp_alu`**：执行算术/逻辑运算，输出 `alu_result` + `cmp_result`
- 实例化 **`rvp_branch_unit`**：判定分支是否跳转，输出 `branch_taken` 和 `branch_target`
- **分析代码中有一处值得注意的注释**：`alu_result_valid_o` 不接 `~flush_i`，因为会导致组合逻辑环路（flush_ex→rf_we_o→ex_reg_write_i→raw_ex_hazard→any_stall→flush_ex）

#### 8. `rvp_mem_stage.sv` — MEM 访存级

**实现方式：**
- **EX/MEM 流水线寄存器**：锁存 ALU 结果、访存控制信号
- **字节使能生成器**：组合逻辑，根据 `mem_size` 和地址低 2 位生成 4 位 `byte_enable`
- **Store 数据对齐**：将 `mem_wdata` 按字节/半字/字移位到正确位置
- **Load 数据对齐与符号扩展**：从 32 位读数据中提取对应字节/半字，符号扩展（lb/lh）或零扩展（lbu/lhu）
- **对齐检查**：字访问时 `addr_offset != 0` 触发 load/store 错误

#### 9. `rvp_wb_stage.sv` — WB 写回级

**实现方式：**
- **MEM/WB 流水线寄存器**：锁存所有写回相关数据
- **写回多路选择**：`unique case` 根据 `wb_src` 从 ALU/MEM/PC4/CSR 中选择
- **x0 保护**：`rf_we_o = rf_we_q & ~flush_i & (rd_addr_q != 5'b0)`

### 三、功能单元

#### 10. `rvp_decoder.sv` — 指令译码器

**实现方式：** 纯组合逻辑，两级 `unique case`：
- **第一级**：按 opcode（7 位）分 8 大类：LUI/AUIPC/JAL/JALR/BRANCH/LOAD/STORE/OP-IMM/OP/SYSTEM
- **第二级**：在需要时按 funct3 + funct7 细分
- 覆盖完整 RV32I 37 条指令，未识别的编码标记 `illegal=1`
- 对每条指令填入 `ctrl_t` 结构体的所有字段

#### 11. `rvp_alu.sv` — 算术逻辑单元

**实现方式：** 纯组合逻辑：
- **加减法**：直接用 `operand_a +/- operand_b`，综合器优化
- **比较逻辑**：`is_equal`（==）、`is_less_signed`（$signed <）、`is_less_unsigned`（<）
- **移位**：`shamt = operand_b[4:0]`，逻辑左移/右移、算术右移（`$signed >>>`）
- **分支结果**：`unique case` 输出 6 种比较（EQ/NE/LT/GE/LTU/GEU），GE 用 `~is_less` 复用
- **主结果**：`unique case` 选择 ADD/SUB/SLL/SRL/SRA/SLT/SLTU/XOR/OR/AND

#### 12. `rvp_register_file.sv` — 寄存器堆

**实现方式：** 32×32 逻辑数组 + 异步读 + 同步写：
- 寄存器阵列：`logic [31:0] regs [31:0]`
- **x0 硬连线为 0**：读时 `raddr==0` 直接返回 0，写时 `waddr==0` 忽略
- **异步读 + 内部前递**：写优先，同周期读写同一寄存器直接返回写数据
- 复位可选（仿真时清零，FPGA 可省略省资源）

#### 13. `rvp_imm_generator.sv` — 立即数生成器

**实现方式：** 纯组合逻辑：
- 先把 5 种格式（I/S/B/U/J）的立即数分别按 RISC-V 手册的位散列规约拼好
- 再用 `unique case` 根据 `imm_type_i` 多路选择输出
- I/S/B/J 型做符号扩展（`{N{inst[31]}}`），U 型直接拼接低 12 位补零

#### 14. `rvp_branch_unit.sv` — 分支判定单元

**实现方式：** 最简单的纯组合逻辑，仅一行：
```systemverilog
assign branch_taken_o = is_branch_i & cmp_result_i;
```
译码器输出 `is_branch_i`，ALU 输出 `cmp_result_i`，两者相与 = 跳转。

#### 15. `rvp_hazard_unit.sv` — 冒险检测单元

**实现方式：** 纯组合逻辑，检测 **Load-Use 冒险**：
- 条件：前一条 ID/EX 指令是 load，且其 rd == 当前 IF/ID 指令的 rs1 或 rs2，且 rd≠0
- 输出 `stall_o` 使 PC 和 IF/ID 停顿，ID/EX 插入 NOP（气泡）

#### 16. `rvp_forward_unit.sv` — 前递单元（条件编译）

**实现方式：** 纯组合逻辑，检测数据冒险并生成前递选择：
- 分别对操作数 A（rs1）和 B（rs2）判断
- **优先级**：EX/MEM 前递（2'b10）> MEM/WB 前递（2'b01）> 不前递（2'b00）
- 前递条件：源级 `reg_write=1`，rd≠0，rd==当前 rs

#### 17. `rvp_controller.sv` — 主控制器

**实现方式：** FSM 状态机（4 个状态）：
- **CTRL_RESET** → **CTRL_FETCH** → 正常取指（PC+4）
- 检测到异常/中断/分支跳转/调试请求 → **CTRL_FLUSH** → 回到 FETCH
- stall 条件 → **CTRL_STALL** 等待
- 处理 ecall（异常号 11）、非法指令（异常号 2）、load/store 对齐错误（异常号 4/6）
- 性能计数：持续统计 `perf_jump` 和 `perf_tbranch`

### 四、Cache 子系统（Phase 3 拓展）

#### 18. `rvp_icache.sv` / `rvp_dcache.sv` — 指令/数据 Cache

**实现方式：** 当前为 **stub 存根**（所有输出信号固定为 0），供编译通过。Cache 使能由 `RVP_ICACHE_ENABLE` / `RVP_DCACHE_ENABLE` 控制。完整实现是 Phase 3 的拓展内容。

#### 19. `rvp_cache_replacement.sv` — 替换策略模块（拓展核心）

**实现方式：** 参数化多策略，通过 `generate if/else if` 条件编译选择：
- **RR (Round-Robin)**：单热旋转指针，每分配一次左移一位
- **LRU (True LRU)**：每路 8 位时间戳计数器，全局 tick 递增，访问时刷新时间戳。victim = 最小时间戳
- **PLRU (Tree-based)**：N-1 位二叉树的"方向"位，从根走到叶选 victim。**更新路径翻转逻辑未完成（TODO）**
- **FIFO**：类似 LRU 但只在新填充时更新时间戳，hit 不刷新
- **RANDOM**：16 位 Galois LFSR 伪随机，零存储开销
- **SRRIP / DRRIP**：保留占位，当前降级为 RR 行为

#### 20. `rvp_cache_tag_array.sv` / `rvp_cache_data_array.sv` / `rvp_cache_stats.sv` / `rvp_cache_flush.sv`

这些模块在项目中列入了文件清单但未实际读取，推测为 Tag/Data 存储阵列、命中率统计计数器和刷写逻辑的辅助模块。

### 五、存储器

#### 21. `rvp_ram_1p.sv` — 单端口 RAM

**实现方式：** 系统级基类，所有存储器模块的底层实现：
- 存储阵列：`logic [31:0] mem [Depth]`
- **字节掩码写**：将 4 位 `be_i` 转为 32 位位掩码，逐字节写入
- **同步读**：1 周期延迟，`rvalid_o = req_i` 延迟一拍
- **固件预加载**：`$readmemh` 支持命令行 `+firmware=` 或参数 `MemInitFile`
- 地址转换：字节地址 → 字索引 `addr_i[Aw-1+2:2]`

#### 22. `rvp_instr_mem.sv` — 指令存储器

**实现方式：** 封装 `rvp_ram_1p` 为只读模式：
- `we=0`、`be=0`、`wdata=0` 固定
- 32KB 容量（`RVP_INSTR_MEM_SIZE=32768`）
- 映射到地址空间 `0x0000_0000`

#### 23. `rvp_data_mem.sv` — 数据存储器

**实现方式：** 封装 `rvp_ram_1p` 为可读写模式：
- 透传 `we/be/wdata` 支持字节级写掩码
- 32KB 容量，映射到 `0x0001_0000`

### 六、外设

#### 24. `rvp_uart.sv` — UART 串口控制器

**实现方式：** 兼容 16550 寄存器子集，波特率可配置：
- **寄存器映射**（5 个偏移）：RBR/THR(0x00)、IER(0x04)、IIR/FCR(0x08)、LCR(0x0C)、LSR(0x10)
- **波特率生成器**：计数器分频 `CLOCK_FREQ/BAUD_RATE`，生成 `baud_tick`
- **TX 状态机**（IDLE→START→DATA→STOP→DONE）：移位发送 8N1 帧
- **RX 状态机**（IDLE→START→DATA→STOP→DONE）：下降沿检测起始位，半波特采样去抖动
- **TX/RX FIFO**：待实例化（TODO），当前代码框架已定义接口
- **中断**：TX 空中断、RX 数据可用中断
- 注意：TX/RX 状态机的核心逻辑目前以注释形式存在（TODO），寄存器读写、波特率生成、中断逻辑已完整

#### 25. `rvp_gpio.sv` — GPIO 控制器

**实现方式：** 16 位可编程 GPIO，已完成实现：
- **寄存器映射**（4 个偏移）：GPIO_IN(0x00, 只读)、GPIO_OUT(0x04)、GPIO_DIR(0x08)、GPIO_IE(0x0C)
- **输入同步**：2 级触发器去亚稳态
- **输出三态**：方向为输出时驱动输出寄存器值，方向为输入时输出高阻 `1'bz`
- **中断逻辑**：待扩展（TODO）

#### 26. `rvp_timer.sv` — 定时器

**实现方式：** 32 位递减计数器，已完成完整实现：
- **寄存器映射**（4 个偏移）：LOAD(0x00)、COUNT(0x04, 只读)、CTRL(0x08)、PRESC(0x0C)
- **预分频器**：每 `presc_q+1` 个时钟周期产生一次 tick
- **递减计数器**：tick 时 count-1，到零后根据模式（单次/自动重装）决定行为
- **中断**：溢出置位 flag，写 1 清除，`irq_o = irq_en & irq_flag`
- 控制寄存器位定义：bit[0]=使能, bit[1]=模式, bit[2]=中断使能, bit[3]=中断标志

### 七、SoC 集成

#### 27. `rvp_bus_interconnect.sv` — 总线互连

**实现方式：** 参数化多主多从互连矩阵：
- **主机仲裁**：固定优先级（编号大者优先），`for` 循环遍历
- **地址译码**：基于掩码匹配 `(addr & mask) == base`，`for` 循环遍历从设备
- **1 周期响应延迟**：请求阶段选择主机/设备，响应阶段路由读数据
- 所有端口均为 `[NrHosts]` / `[NrDevices]` 的 unpacked 数组

#### 28. `rvp_soc_top.sv` — SoC 顶层

**实现方式：** 纯结构化的顶层集成模块：
- 实例化 `rvp_core` + `rvp_instr_mem` + `rvp_bus_interconnect` + `rvp_data_mem` + `rvp_uart` + `rvp_gpio` + `rvp_timer`
- **哈佛结构**：指令存储器直接连 CPU（不经过总线），数据通路经总线互连
- **地址映射**：

| 设备 | 基地址 | 大小 |
|---|---|---|
| 指令存储器 | `0x0000_0000` | 32KB |
| 数据存储器 | `0x0001_0000` | 32KB |
| UART | `0x1000_0000` | 4KB |
| GPIO | `0x1001_0000` | 4KB |
| Timer | `0x1002_0000` | 4KB |

- UART 和 GPIO 用 `` `ifdef RVP_UART_ENABLE `` / `` `ifdef RVP_GPIO_ENABLE `` 条件编译
