## RISC-V RV32I + M 指令集一览

### 一、U 型指令（高位立即数操作）

| 指令 | 类型 | 功能说明 |
|------|------|---------|
| **lui** | U型 | 把一个大数（20位）放到寄存器的高20位，低12位自动填0。例如 `lui x5, 0x12345` → x5=0x12345000 |
| **auipc** | U型 | 把当前PC的值加上一个大数，结果存到寄存器。通常和 `jalr` 配合做远距离函数调用 |

> **必要性分析**
>
> - **lui**：**绝对必要。** 这是 RISC-V 中构建 32 位大立即数的唯一方式。外设地址（如 UART 的 `0x10000000`）必须用 `lui` 将高 20 位置入寄存器，再配合 `addi` 或 `sw` 完成 MMIO 访问。没有 `lui`，外设编程无法进行。
> - **auipc**：**必要。** 标准 RISC-V ABI 要求 PC 相对寻址来生成位置无关代码。在加载全局变量地址、跳转表等场景中使用。虽然理论上可被 `lui + jalr` 组合等效替代，但代码膨胀且不符合 ABI 约定，编译器会依赖它。

---

### 二、跳转指令

| 指令       | 类型  | 功能说明                                         |
| -------- | --- | -------------------------------------------- |
| **jal**  | J型  | 函数调用：把"下一条指令的地址"（返回地址）保存到寄存器，然后跳到目标地址        |
| **jalr** | I型  | 函数返回/间接跳转：把返回地址保存到寄存器，然后跳到 "寄存器的值+一个小偏移" 的位置 |

> **必要性分析**
>
> - **jal**：**绝对必要。** 唯一的直接函数调用指令。任何程序中只要有函数调用（C 代码的 `func()`、测试程序的 `RVTEST_PASS` 宏），都依赖它保存返回地址并跳转。没有 `jal`，程序无法组织成模块化的函数结构。
> - **jalr**：**绝对必要。** 唯一的函数返回指令（`jalr x0, ra, 0` 即 `ret`）。此外也是间接跳转（如 switch-case 的跳转表）的唯一方式。没有它，函数调用后无法返回，程序无法运行。

---

### 三、条件分支指令

| 指令 | 类型 | 功能说明 |
|------|------|---------|
| **beq** | B型 | 如果两个寄存器相等，就跳转到目标地址（if rs1 == rs2） |
| **bne** | B型 | 如果两个寄存器不相等，就跳转（if rs1 != rs2） |
| **blt** | B型 | 如果有符号比较 rs1 < rs2，就跳转（有符号小于） |
| **bge** | B型 | 如果有符号比较 rs1 >= rs2，就跳转（有符号大于等于） |
| **bltu** | B型 | 如果无符号比较 rs1 < rs2，就跳转（无符号小于） |
| **bgeu** | B型 | 如果无符号比较 rs1 >= rs2，就跳转（无符号大于等于） |

> **必要性分析**
>
> - **beq、bne**：**绝对必要。** 任何条件逻辑的基础。if-else、循环终止判断、测试程序的 PASS/FAIL 比较都依赖相等/不等判断。测试宏 `bne t2, t3, fail` 是每个测试用例的核心。
> - **blt、bge**：**必要。** for 循环的 `i < n`（blt）和 `i >= 0`（bge）直接映射到这两条指令。没有它们，所有有符号大小比较都要拆成 beq/bne + 额外算术指令，代码量翻倍且可读性差。
> - **bltu、bgeu**：**必要。** 用于无符号地址范围检查。例如 MMIO 编程中需要判断地址是否超出外设地址空间（`0x10000000 ~ 0x1002FFFF`），必须用无符号比较，因为地址作为 32 位无符号数，有符号比较会出错。六条分支复用 ALU 中同一套比较器（`is_equal`、`is_less_signed`、`is_less_unsigned`），bge 用 `~is_less_signed`、bgeu 用 `~is_less_unsigned`，不增加硬件。

### 四、读内存指令（Load）

| 指令 | 类型 | 功能说明 |
|------|------|---------|
| **lw** | I型 | 从内存地址 "rs1+偏移量" 读出32位数据，存入寄存器 |
| **lh** | I型 | 从内存读出16位数据，把最高位复制到高16位（符号扩展），存入寄存器 |
| **lhu** | I型 | 从内存读出16位数据，高16位填0（零扩展），存入寄存器 |
| **lb** | I型 | 从内存读出8位数据，把最高位复制到高24位（符号扩展），存入寄存器 |
| **lbu** | I型 | 从内存读出8位数据，高24位填0（零扩展），存入寄存器 |

> **必要性分析**
>
> - **lw**：**绝对必要。** 32 位字加载是 MMIO 访问的唯一方式。外设寄存器（UART 状态、GPIO 输入、Timer 计数值）都是 32 位映射，必须用 `lw` 读取。此外所有内存变量访问也依赖它。data_mem 和总线互连都是 32 位宽。
> - **lh、lhu**：**基本必要。** 16 位有符号/无符号加载。处理 16 位外设寄存器域时使用。即使当前测试程序以 32 位为主，保留它们成本极低——MEM 级的字节使能生成和符号扩展逻辑只需在 `mem_size=SIZE_H` 分支下多两个符号/零扩展选择，综合器优化后几乎无额外逻辑。
> - **lb、lbu**：**基本必要。** 8 位有符号/无符号加载。字符串处理（如测试程序向 UART 输出 "OK\n" 时逐字符读取）需要字节级访问。此外 picorv32 测试框架的多条测试用例（`lb.S`、`lbu.S`）依赖字节加载。

### 五、写内存指令（Store）

| 指令 | 类型 | 功能说明 |
|------|------|---------|
| **sw** | S型 | 把寄存器中32位数据写到内存地址 "rs1+偏移量" 处 |
| **sh** | S型 | 把寄存器的低16位写到内存地址处 |
| **sb** | S型 | 把寄存器的低8位写到内存地址处 |

> **必要性分析**
>
> - **sw**：**绝对必要。** 32 位字存储是外设控制的核心方式。向 UART 发送寄存器的 RISC-V 汇编本质就是 `sw t0, 0x10000000(x0)`。向 GPIO 输出寄存器写 LED 值、向 Timer 加载值寄存器写计数初始值，全都依赖 `sw`。此外所有内存变量写入也依赖它。
> - **sh**：**基本必要。** 16 位存储。与 `lh`/`lhu` 配套，用于 16 位外设寄存器域写入。MEM 级的 Store 数据对齐逻辑与 `sw` 共享基础框架。
> - **sb**：**基本必要。** 8 位存储。UART 逐字符发送（"O"→"K"→"\n"）在 RISC-V 汇编中就是 `sb` 写入 THR 寄存器。picorv32 测试框架的 `RVTEST_PASS` 宏正是这样输出测试结果的。MEM 级字节对齐与 `lb`/`lbu` 对称实现。

### 六、立即数算术指令（OP-IMM）

| 指令 | 类型 | 功能说明 |
|------|------|---------|
| **addi** | I型 | 寄存器 + 小常数，结果存入寄存器。例如 `addi x5, x6, 100` → x5 = x6 + 100 |
| **slti** | I型 | 有符号比较：如果 寄存器 < 小常数，结果=1，否则=0 |
| **sltiu** | I型 | 无符号比较：如果 寄存器 < 小常数，结果=1，否则=0 |
| **xori** | I型 | 寄存器 按位异或 小常数 |
| **ori** | I型 | 寄存器 按位或 小常数 |
| **andi** | I型 | 寄存器 按位与 小常数 |
| **slli** | I型 | 寄存器左移N位（N是小常数的低5位），低位补0 |
| **srli** | I型 | 寄存器逻辑右移N位，高位补0 |
| **srai** | I型 | 寄存器算术右移N位，高位补原来的符号位（保持正负不变） |

> **必要性分析**
>
> - **addi**：**绝对必要。** 使用频率最高的指令。栈指针加减、循环计数器递增、地址偏移计算（`lw rd, offset(rs1)` 中的 offset）、以及 RISC-V 中的 nop（`addi x0, x0, 0`）都依赖它。没有 addi，每条算术运算都需要先 lui 加载常数再 add，指令数翻倍。
> - **slti、sltiu**：**必要。** 立即数与寄存器的有符号/无符号比较。循环条件 `i < 100` 中被编译为 `slti + bne` 组合。由于 ALU 的比较器（`is_less_signed`/`is_less_unsigned`）已为分支指令存在，这两条只是多了一个译码分支把比较结果存回寄存器，硬件成本为零。
> - **xori**：**必要。** 立即数异或用于位翻转、寄存器清零（`xori rd, rs1, -1` = 取反）、特定位置反。在 GPIO 编程中切换 LED 状态时需要。ALU 异或门已为 `xor` 指令存在。
> - **ori**：**必要。** 立即数或用于位置位。外设寄存器配置时，设置控制位（如 Timer CTRL 的 bit[0]=使能）必须用 `ori`。此外 `ori rd, rs1, 0` 可作为寄存器间移动（mv 伪指令）。
> - **andi**：**必要。** 立即数与用于位掩码提取。读取外设状态寄存器后，用 `andi` 提取特定状态位（如 UART LSR[0] 判断接收数据就绪）是外设驱动的标准操作。
> - **slli**：**必要。** 左移 = 乘 2^N。字节地址转字索引（`addr >> 2` 本质是右移，但配合 `slli` 做对齐计算）。例如数组索引 `arr[i]` 的地址计算 `base + i*4` 编译为 `slli + add`。
> - **srli**：**必要。** 逻辑右移用于提取位域。例如从 UART 读回的三字节数据 `0x00_00_00_4F` 中提取 'O' 字符。ALU 右移路径已为 `srl` 指令存在。
> - **srai**：**必要。** 算术右移 = 有符号除 2^N（保持符号位）。ALU 的 `$signed >>>` 路径已为 `sra` 指令存在，译码器多一行。

### 七、寄存器间算术指令（OP）

| 指令 | 类型 | 功能说明 |
|------|------|---------|
| **add** | R型 | 两寄存器相加 |
| **sub** | R型 | 两寄存器相减 |
| **sll** | R型 | 左移：rs1 << rs2的低5位 |
| **slt** | R型 | 有符号小于比较：rs1<rs2?1:0 |
| **sltu** | R型 | 无符号小于比较：rs1<rs2?1:0 |
| **xor** | R型 | 按位异或 |
| **srl** | R型 | 逻辑右移：高位补0 |
| **sra** | R型 | 算术右移：高位补符号位 |
| **or** | R型 | 按位或 |
| **and** | R型 | 按位与 |

> **必要性分析**
>
> 全部 10 条 OP 指令均为必要，原因与其对应的 OP-IMM 版本一致，只是操作数 B 来自 rs2 而非立即数。要点：
> - **add、sub**：**绝对必要。** 寄存器间加减是整个指令集的核心运算。`add` 用于地址计算（rs1+rs2 的变址寻址）、指针加法；`sub` 用于比较的底层实现。ALU 的加减法路径是所有运算的基础。
> - **sll、srl、sra**：**必要。** 变量移位（移位量在寄存器中而非立即数）。例如循环中根据计数器做动态移位、乘法用移位-加法实现时都需要变量移位。与 I 型移位指令共享 ALU 的移位路径。
> - **slt、sltu**：**必要。** 两个寄存器的大小比较置位。测试程序的 `bne t2, t3, fail` 前常有 `slt` 把比较结果写入 t2。与分支比较共享 ALU 的比较器。
> - **xor、or、and**：**必要。** 寄存器间按位逻辑运算。位字段操作、数据掩码、外设寄存器的位操作都依赖这三条。与 I 型逻辑指令共享 ALU 门电路。

### 八、乘除法指令（M扩展）

| 指令         | 类型  | 功能说明                          |
| ---------- | --- | ----------------------------- |
| **mul**    | R型  | 有符号乘法，取结果的低32位                |
| **mulh**   | R型  | 有符号乘法，取结果的高32位（用于判断溢出或做64位运算） |
| **mulhsu** | R型  | 混合乘法：rs1有符号 × rs2无符号，取高32位    |
| **mulhu**  | R型  | 无符号乘法，取高32位                   |
| **div**    | R型  | 有符号除法 rs1 ÷ rs2；除0时结果为-1      |
| **divu**   | R型  | 无符号除法 rs1 ÷ rs2；除0时结果为全1      |
| **rem**    | R型  | 有符号取余 rs1 % rs2；除0时结果=rs1     |
| **remu**   | R型  | 无符号取余 rs1 % rs2；除0时结果=rs1     |

> **必要性分析**
>
> M 扩展已确认为本项目必须实现。
>
> - **mul**：**必要。** 基本乘法。属于"算术运算"基本类型。任何乘常数、数组索引 `i*sizeof(element)` 的编译器优化都依赖乘法指令。实现方式：32×32→64 乘法器，取低 32 位。
> - **mulh、mulhsu、mulhu**：**必要。** 高半乘法。用于 64 位乘法合成（`mul` 取低 32 位 + `mulh` 取高 32 位拼成 64 位结果）、溢出检测（`mulh` 结果非全 0 或全 1 表示溢出）。三条指令复用同一个 32×32→64 乘法器，根据操作数符号选择不同的符号扩展方式，不增加额外乘法器硬件。
> - **div**：**必要。** 有符号除法，基本的算术运算。实现方式：多周期除法器（如 32 周期移位-减法算法），在 EX 级引入 stall 直到除法完成。除 0 时结果 = -1（RISC-V 规范）。
> - **divu**：**必要。** 无符号除法。与 `div` 共享同一套除法器硬件，仅操作数解读方式不同。除 0 时结果 = 全 1。
> - **rem**：**必要。** 有符号取余。与 `div` 共享同一套除法器——除法器在计算商的同时自然得到余数。除 0 时结果 = 被除数。
> - **remu**：**必要。** 无符号取余。与 `divu` 共享除法器。

---

共 **45 条指令**，其中 RV32I 基础 37 条 + M 扩展 8 条。每条指令编码中实际上还隐含了 **nop**（即 `addi x0, x0, 0`，虽然被译码器正常译码，但因为目标寄存器是 x0 所以写操作无效，等价于空操作）。
