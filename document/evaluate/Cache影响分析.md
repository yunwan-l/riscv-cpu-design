# Cache 影响分析

## 1. 当前存储系统架构

### 1.1 存储器配置

RVP SoC 当前使用直接寻址的片上存储器，无 Cache 层级：

| 存储器 | 容量 | 访问方式 | 延迟 | 实现方式 |
|--------|------|----------|------|----------|
| 指令存储器（IMEM） | 8 KB（2048 字） | 异步读、同步写 | 1 周期 | 分布式 RAM / LUT |
| 数据存储器（DMEM） | 8 KB（2048 字） | 异步读、同步写 | 1 周期 | 分布式 RAM / LUT |

### 1.2 访问特性

当前存储器的关键特性：

- **异步读**：地址在时钟上升沿有效后，数据在同一周期内立即可用，无需等待周期
- **单周期访问**：Load 指令在 MEM 阶段读取数据，数据在当周期即可前递到 EX 阶段的下一条指令（需 1 周期 Load-Use 停顿）
- **无缺失惩罚**：所有访问均命中，等效于 100% 命中率的完美 Cache

### 1.3 等效模型

当前存储系统可等效为：

```
完美 Cache 模型：
  命中率 = 100%
  命中延迟 = 1 周期
  缺失延迟 = N/A（无缺失）
  平均访问时间 = 1 周期
```

## 2. 加入 I-Cache（指令 Cache）的影响分析

### 2.1 I-Cache 架构假设

| 参数 | 值 | 说明 |
|------|-----|------|
| Cache 容量 | 4 KB | 典型 L1 I-Cache 大小 |
| 块大小 | 32 字节（8 条指令） | 每次缺失加载 8 条指令 |
| 关联度 | 直接映射 | 简单硬件实现 |
| 命中延迟 | 1 周期 | 与当前存储器相同 |
| 缺失惩罚 | 8 周期 | 从后备存储加载 8 条指令的延迟 |

### 2.2 基准测试程序的 I-Cache 命中率分析

#### 矩阵乘法（76 条指令，304 字节）

| 特征 | 分析 |
|------|------|
| 程序总大小 | 304 字节 < 4 KB Cache 容量 |
| 循环结构 | 三重嵌套循环，内层循环体约 20 条指令 |
| 空间局部性 | 顺序取指，Cache 行（32 字节 = 8 条指令）覆盖率高 |
| 时间局部性 | 内层循环每次迭代执行相同指令序列 |
| 预估命中率 | > 99.9% |
| 缺失次数 | 程序启动时约 10 次（304/32 ≈ 10 行），之后循环全部命中 |
| I-Cache 对 CPI 影响 | +0.015（10 次缺失 × 8 周期 / 674 指令） |

#### 冒泡排序（44 条指令，176 字节）

| 特征 | 分析 |
|------|------|
| 程序总大小 | 176 字节 < 4 KB Cache 容量 |
| 循环结构 | 双重循环，内层循环体约 12 条指令 |
| 预估命中率 | > 99.9% |
| 缺失次数 | 程序启动时约 6 次（176/32 ≈ 6 行） |
| I-Cache 对 CPI 影响 | +0.012（6 次缺失 × 8 周期 / 482 指令） |

#### 斐波那契（15 条指令，60 字节）

| 特征 | 分析 |
|------|------|
| 程序总大小 | 60 字节 < 4 KB Cache 容量 |
| 循环结构 | 单循环，循环体约 8 条指令 |
| 预估命中率 | > 99.9% |
| 缺失次数 | 程序启动时约 2 次（60/32 ≈ 2 行） |
| I-Cache 对 CPI 影响 | +0.188（2 次缺失 × 8 周期 / 85 指令） |

### 2.3 I-Cache 影响总结

| 程序 | 当前 CPI | 加 I-Cache 后 CPI | CPI 变化 | 说明 |
|------|----------|-------------------|----------|------|
| 矩阵乘法 | 1.0846 | 1.0996 | +0.015 (+1.4%) | 缺失惩罚极小 |
| 冒泡排序 | 1.1826 | 1.1946 | +0.012 (+1.0%) | 缺失惩罚极小 |
| 斐波那契 | 1.1529 | 1.3409 | +0.188 (+16.3%) | 短程序冷启动缺失占比大 |

**结论**：对于小规模基准测试程序，加入 I-Cache 反而会略微增加 CPI（冷启动缺失惩罚），因为当前直接寻址存储器已经等效于 100% 命中的完美 Cache。I-Cache 的优势在大程序（超出片上存储容量）场景下才能体现。

## 3. 加入 D-Cache（数据 Cache）的影响分析

### 3.1 D-Cache 架构假设

| 参数 | 值 | 说明 |
|------|-----|------|
| Cache 容量 | 4 KB | 典型 L1 D-Cache 大小 |
| 块大小 | 32 字节（8 个字） | 每次缺失加载 8 个字 |
| 关联度 | 2 路组相联 | 平衡命中率和硬件复杂度 |
| 写策略 | 写直达（Write-Through） | 简化一致性管理 |
| 命中延迟 | 1 周期 | 与当前存储器相同 |
| 缺失惩罚 | 8 周期 | 从后备存储加载 8 个字的延迟 |

### 3.2 基准测试程序的 D-Cache 访问模式分析

#### 矩阵乘法

| 访问模式 | 分析 |
|----------|------|
| 数据总量 | A(36B) + B(36B) + C(36B) = 108 字节 < 4 KB |
| 访问序列 | 内层循环：lw A[i][k], lw B[k][j], sw C[i][j] |
| 空间局部性 | 矩阵按行存储，内层 k 递增时 A[i][k] 连续访问 |
| 时间局部性 | A 和 B 在不同 i,j 迭代中被重复访问 |
| Load 指令数 | 27 次 lw A + 27 次 lw B = 54 次 |
| 预估命中率 | > 98%（数据量远小于 Cache） |
| 缺失次数 | 首次加载 A(5 行) + B(5 行) + C(5 行) ≈ 15 次 |
| D-Cache 对 CPI 影响 | +0.178（15 次缺失 × 8 周期 / 674 指令） |

#### 冒泡排序

| 访问模式 | 分析 |
|----------|------|
| 数据总量 | 8 个元素 = 32 字节 < 4 KB |
| 访问序列 | 内层循环：lw arr[j], lw arr[j+1], sw arr[j], sw arr[j+1] |
| 空间局部性 | 连续访问相邻元素 arr[j] 和 arr[j+1] |
| 时间局部性 | 同一元素在多次内层迭代中被访问 |
| Load 指令数 | 每次内层迭代 2 次 lw，共约 56 次 |
| 预估命中率 | > 99%（数据量仅 32 字节 = 1 个 Cache 行） |
| 缺失次数 | 首次加载数组 ≈ 1 次 |
| D-Cache 对 CPI 影响 | +0.017（1 次缺失 × 8 周期 / 482 指令） |

#### 斐波那契

| 访问模式 | 分析 |
|----------|------|
| 数据总量 | 10 个字 = 40 字节 < 4 KB |
| 访问序列 | 每次循环：sw fib[counter] |
| 空间局部性 | 顺序写入连续地址 |
| Load 指令数 | 0 次 lw（纯写程序） |
| 预估命中率 | N/A（无 Load 操作） |
| D-Cache 对 CPI 影响 | +0.000（无数据读取缺失） |

### 3.3 D-Cache 影响总结

| 程序 | 当前 CPI | 加 D-Cache 后 CPI | CPI 变化 | 说明 |
|------|----------|-------------------|----------|------|
| 矩阵乘法 | 1.0846 | 1.263 | +0.178 (+16.4%) | 冷启动缺失较多 |
| 冒泡排序 | 1.1826 | 1.199 | +0.017 (+1.4%) | 数据量极小 |
| 斐波那契 | 1.1529 | 1.153 | +0.000 (+0.0%) | 无 Load 操作 |

**结论**：对于小规模数据集，D-Cache 的主要开销来自冷启动缺失（首次加载数据时的强制缺失）。矩阵乘法受影响最大（+16.4% CPI），因为其数据量最大且访问模式更复杂。冒泡排序和斐波那契几乎不受影响。

## 4. 同时加入 I-Cache + D-Cache 的综合影响

### 4.1 综合 CPI 影响

| 程序 | 当前 CPI | 加 Cache 后 CPI | CPI 变化 | I-Cache 贡献 | D-Cache 贡献 |
|------|----------|----------------|----------|-------------|-------------|
| 矩阵乘法 | 1.0846 | 1.278 | +0.193 (+17.8%) | +0.015 | +0.178 |
| 冒泡排序 | 1.1826 | 1.211 | +0.029 (+2.4%) | +0.012 | +0.017 |
| 斐波那契 | 1.1529 | 1.341 | +0.188 (+16.3%) | +0.188 | +0.000 |

### 4.2 分析结论

对于当前的小规模基准测试程序，加入 Cache 会**增加** CPI 而非降低，原因如下：

1. **当前存储器已是"完美 Cache"**：1 周期异步读等效于 100% 命中率的 L1 Cache，任何实际 Cache 的命中率都低于 100%
2. **程序和数据量极小**：所有基准测试程序（最大 304 字节代码 + 108 字节数据）远小于 4 KB Cache 容量，但 Cache 的冷启动缺失仍会引入额外周期
3. **缺失惩罚不可忽略**：即使缺失率很低（< 2%），每次缺失 8 周期的惩罚对 CPI 的影响仍超过 10%

### 4.3 Cache 有益的场景

Cache 在以下场景中才能带来正面收益：

| 场景 | 当前存储器 | Cache 方案 | Cache 优势 |
|------|-----------|-----------|-----------|
| 大程序（>8KB） | 无法容纳 | I-Cache + 外部 Flash | 命中时 1 周期 vs 外部 10+ 周期 |
| 大数据集（>8KB） | 无法容纳 | D-Cache + 外部 SRAM | 命中时 1 周期 vs 外部 5+ 周期 |
| 外部 DRAM 后备 | 50-100 周期 | L1 Cache + DRAM | 命中时 1 周期 vs DRAM 50+ 周期 |
| 多任务切换 | 每次全加载 | Cache + 替换策略 | 热数据保留在 Cache 中 |

## 5. 不同存储层级方案对比

### 5.1 方案对比

| 方案 | 命中延迟 | 缺失延迟 | 平均 CPI 贡献 | 适用场景 |
|------|----------|----------|-------------|----------|
| 当前（无 Cache） | 1 周期 | N/A | 1.000 | 小程序，数据量 < 8KB |
| L1 Cache + BRAM 后备 | 1 周期 | 2-3 周期 | 1.001-1.010 | 中等程序，偶尔缺失 |
| L1 Cache + 外部 SRAM | 1 周期 | 5-10 周期 | 1.050-1.150 | 大程序，频繁缺失 |
| L1 Cache + 外部 DRAM | 1 周期 | 50-100 周期 | 1.500-2.000 | 超大程序，高缺失率 |
| L1 + L2 Cache | 1 周期 | L2: 5-10 周期 | 1.010-1.030 | 大程序，低缺失率 |

### 5.2 推荐方案

对于当前 RVP SoC 的典型应用场景（嵌入式小规模程序）：

- **当前方案（无 Cache）已最优**：程序和数据量均小于片上存储容量，1 周期访问无缺失
- **未来扩展**：若程序规模增长超过 8KB，建议添加 I-Cache（4KB 直接映射 + BRAM 后备），缺失惩罚仅 2-3 周期
- **不建议添加 D-Cache**：数据存储器使用 BRAM 直接实现已足够，D-Cache 的写策略复杂性不值得为小数据集引入

---

## 6. 总结

### 6.1 关键发现

1. **当前无 Cache 设计对小规模程序最优**：1 周期异步读等效于完美 Cache，无缺失惩罚
2. **加入 Cache 会增加 CPI 1.4%~17.8%**：冷启动缺失的额外周期无法被低缺失率抵消
3. **I-Cache 对短程序影响大**：斐波那契仅 15 条指令，2 次冷启动缺失导致 +16.3% CPI
4. **D-Cache 对大数据量影响大**：矩阵乘法 108 字节数据，15 次冷启动缺失导致 +16.4% CPI
5. **Cache 的价值在规模化场景**：当程序或数据超过片上存储容量时，Cache 的平均访问时间优势才能体现

### 6.2 对 CPI 的量化影响

| 方案 | 矩阵乘法 CPI | 冒泡排序 CPI | 斐波那契 CPI | 平均 CPI |
|------|-------------|-------------|-------------|----------|
| 当前（无 Cache） | 1.0846 | 1.1826 | 1.1529 | 1.1400 |
| +I-Cache | 1.0996 (+1.4%) | 1.1946 (+1.0%) | 1.3409 (+16.3%) | 1.2117 (+6.3%) |
| +D-Cache | 1.263 (+16.4%) | 1.199 (+1.4%) | 1.153 (+0.0%) | 1.205 (+5.7%) |
| +I-Cache +D-Cache | 1.278 (+17.8%) | 1.211 (+2.4%) | 1.341 (+16.3%) | 1.277 (+12.0%) |

> 注：以上 Cache 影响为理论估算值，基于冷启动强制缺失（compulsory miss）模型，未考虑容量缺失和冲突缺失。实际命中率可能因 Cache 组织方式和替换策略而有所不同。
