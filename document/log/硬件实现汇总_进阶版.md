# RVP RISC-V 处理器硬件实现汇总（进阶版）

> 完成日期：2026-07-10  
> 开发板：Nexys4 DDR（Xilinx Artix-7 XC7A100T）  
> 工具链：Vivado 2018.3 + 自研 rv_assembler.py

---

## 一、项目概述

本项目实现了一个支持 RV32IM 指令集的 5 级流水线 RISC-V 处理器，在 Nexys4 DDR FPGA 开发板上完成综合、实现和板载验证。处理器包含 I-Cache、流水线前递、Load-Use 冒险检测、M 扩展乘除法单元、GPIO/UART/Timer 外设，并通过拨码开关实现了多程序集成切换。

---

## 二、硬件架构

### 2.1 时钟与分频

| 时钟域 | 频率 | 用途 |
|--------|------|------|
| 板载晶振 | 100 MHz | 输入时钟（E3 引脚） |
| clk_soc | 25 MHz | CPU + SoC（4 分频，周期 40ns） |
| clk_100 | 100 MHz | 数码管扫描（1 kHz 刷新） |

分频通过 2 位计数器实现，`clk_soc = clk_cnt[1]`，占空比 50%。

### 2.2 CPU 核心设计

**5 级流水线：** IF → ID → EX → MEM → WB

| 模块 | 文件 | 功能 |
|------|------|------|
| 取指 | `rvp_core_pipeline.sv` | PC 生成、I-Cache 接口 |
| 译码 | `rvp_decoder.sv` | RV32IM 指令译码，生成控制信号 |
| 执行 | `rvp_alu.sv` | 10 种 ALU 运算（ADD/SUB/SLL/SRL/SRA/SLT/XOR/OR/AND） |
| 乘除法 | `rvp_multdiv.sv` | M 扩展多周期乘除法（MUL/MULH/DIV/DIVU/REM/REMU） |
| 访存 | `rvp_data_mem.sv` | 字/半字/字节读写，符号扩展 |
| 写回 | `rvp_core_pipeline.sv` | 寄存器堆写回 |
| 前递 | `rvp_forward_unit.sv` | EX/MEM 和 MEM/WB 数据前递 |
| 冒险 | `rvp_hazard_unit.sv` | Load-Use 冒险检测，1 周期停顿 |
| 流水线寄存器 | `rvp_pipeline_regs.sv` | IF/ID、ID/EX、EX/MEM、MEM/WB 四组寄存器 |

**分支处理：** 分支在 EX 阶段解算，成立时冲刷 IF/ID 和 ID/EX（2 个气泡）。

### 2.3 I-Cache

| 参数 | 值 |
|------|-----|
| 结构 | 直接映射（Direct-Mapped） |
| 缓存行数 | 64 行 |
| 每行大小 | 1 字（32 位） |
| 总容量 | 256 字节 |
| 索引 | PC[7:2]（6 位） |
| Tag | PC[31:8]（24 位） |
| 读方式 | 异步读（命中和未命中均同周期返回） |
| 填充方式 | 未命中时同步填充 |
| 统计 | 命中/未命中计数器 |

### 2.4 外设

| 外设 | 地址范围 | 寄存器 |
|------|---------|--------|
| Data RAM | 0x00000000 ~ 0x00001FFF | 2048 字（8KB），异步读 |
| UART | 0x10000000 ~ 0x1000FFFF | TXDATA(W, 0x00), TXSTAT(R, 0x04, bit0=busy) |
| GPIO | 0x10010000 ~ 0x1001FFFF | OUTPUT(RW, 0x00), INPUT(R, 0x04, SW[15:0]) |
| Timer | 0x10020000 ~ 0x1002FFFF | 32 位定时器 |
| Perf | 0x10030000 ~ 0x1003FFFF | 性能计数器（cycle/inst/stall/flush/branch） |

**UART 参数：** 波特率 115200，8N1，CLK_FREQ=25MHz，分频系数=217。

**地址译码：** `dbus_addr[31:16]` 选择外设。

### 2.5 FPGA 资源利用

| 资源 | 使用量 | 占比 |
|------|--------|------|
| LUT | 4,998 | 7.88% |
| FF | 5,477 | 4.32% |
| BRAM | 1 | - |
| DSP | 12 | 5.00% |
| WNS | +6.247 ns | - |
| Fmax | 29.63 MHz | - |

## 三、固件系统

### 3.1 集成固件（firmware_all.hex）

**总大小：** 1244/2048 字（60.7%）

通过拨码开关 SW[3:0] 选择运行程序，按 RESET 切换，无需重新综合。

| SW[3:0] | 测试 | 入口地址 | 大小 | 验证功能 |
|---------|------|---------|------|---------|
| 0000 | LED Blink | 0x070 | 15 字 | 基础流水线连通性 |
| 0001 | Counter | 0x0AC | 10 字 | ALU 加法 + 寄存器写回 |
| 0010 | Forward | 0x0D4 | 25 字 | EX/MEM 数据前递（11 级链） |
| 0011 | Load-Use | 0x138 | 17 字 | Load-Use 冒险停顿 |
| 0100 | Branch | 0x17C | 389 字 | BEQ/BNE/BLT/BGE/JAL |
| 0101 | ALU | 0x790 | 309 字 | 9 种 ALU 运算 |
| 0110 | Memory | 0xC64 | 216 字 | SW/LW/SB/LB/LBU/SH/LH 符号扩展 |
| 0111 | Muldiv | 0xFC4 | 235 字 | MUL/MULH/DIV/DIVU/REM/REMU |

### 3.2 程序切换机制

```
地址 0x00: lui x5, 0x10010       # GPIO 基地址
地址 0x04: lw x6, 4(x5)          # 读取 SW[15:0]
地址 0x08: andi x6, x6, 0xF      # 取 SW[3:0]
地址 0x0C~0x48: BEQ 分发跳转      # 按 SW 值跳转到对应测试入口
```

无需修改 RTL，GPIO INPUT 寄存器天然支持读取拨码开关。

## 五、开发流程

### 5.1 固件开发

1. 编写汇编代码（使用 `rv_assembler.py` 汇编器）
2. 汇编器自动插入 NOP 延迟槽（CPU 实际冲刷，无害）
3. 生成 hex 文件，复制到 3 个位置：
   - `C:\rvp_proj\firmware.hex`
   - `C:\rvp_proj\synth\vivado\firmware.hex`
   - `C:\rvp_proj\build\vivado\rvp_nexys4.srcs\sources_1\imports\firmware.hex`
4. Vivado 中 `reset_run synth_1` → 重新综合 → 实现 → 生成比特流

### 5.2 板载测试

1. Vivado Hardware Manager → Program Device
2. 设置拨码开关 SW[3:0] 选择测试程序
3. 按 RESET 键启动
4. LED 测试：肉眼观察
5. UART 测试：MobaXterm Serial 会话（115200, 8N1）

### 5.3 关键经验

- **Vivado 固件缓存：** `$readmemh` 在 `initial` 块中执行，综合时读取 hex 文件。必须同时更新所有 3 个 `firmware.hex` 副本，否则 Vivado 可能读到旧文件。
- **\r\n 换行：** UART 发送文本需要 `\r\n`（0x0D+0x0A），仅 `\n` 会导致终端对角线缩进。
- **I-Cache 冷启动：** 切换程序后产生 64 次冷缺失，但不影响功能正确性。

## 七、已知限制与未来改进

1. **UART 仅支持发送（TX-only）：** 无接收功能，无法通过电脑发送命令控制 CPU
2. **I-Cache 仅 256 字节：** 循环体较大的程序命中率有限
3. **无 D-Cache：** 所有数据访问直接访问 BRAM
4. **无中断/异常处理：** 不支持 CSR 指令和中断响应
5. **UART 波特率固定 115200：** 如需降低可修改 `rvp_soc.sv` 中的 `BAUD_RATE` 参数
